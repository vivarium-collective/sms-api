# Plan: Remove PCS/SLURM and FSx from stanford-test Deployment

## Context

The `sms-api-stanford-test` namespace uses the K8s/AWS Batch backend for simulations. SLURM, SSH, and FSx infrastructure remain configured but are unused. This plan removes them cleanly.

## Phase 1: Kustomize Cleanup (Safe, No Code Changes)

Remove from `kustomize/overlays/sms-api-stanford-test/`:

| File | What It Is | Why Safe to Remove |
|------|------------|-------------------|
| `fsx-pcs-root-pv.yaml` | FSx Lustre PersistentVolume | K8s backend writes to S3, not FSx |
| `fsx-pcs-root-pv.yaml.template` | Template for above | Same |
| `vivarium-home-pvc.yaml` | PVC bound to FSx PV | Not used by any K8s-backend code path |
| `storageclass-gp3-retain.yaml` | StorageClass for EBS | Only needed by FSx PV |
| `secret-ssh.yaml` | SSH key for SLURM login node | K8s backend doesn't SSH anywhere |

Update `kustomization.yaml`:
- Remove resources: `fsx-pcs-root-pv.yaml`, `vivarium-home-pvc.yaml`, `storageclass-gp3-retain.yaml`, `secret-ssh.yaml`
- Remove the `vivarium-home-pvc` volumeMount patch (lines 35-44)
- Remove `haproxy-ssh` from replicas (already 0, just remove the entry)

Update `kustomize/base/api.yaml`:
- The base defines SSH key and known_hosts volume mounts. For stanford-test, add a patch that removes these mounts. Or refactor the base to make them optional.

## Phase 3: Code Fixes (Required for Clean Deployment)

### 3a. Analysis Endpoint — Backend Guard

**File:** `sms_api/api/routers/gateway.py`

The `/analyses` POST endpoint unconditionally creates `AnalysisServiceSlurm`. Add backend check:

```python
if get_job_backend() == "slurm":
    analysis_service = AnalysisServiceSlurm(env=ENV)
    return await handlers.analyses.handle_run_analysis(...)
else:
    raise HTTPException(501, "Analysis submission not yet supported for K8s backend")
```

### 3b. Parca Endpoint — Backend Guard

**File:** `sms_api/api/routers/core.py`

The `/simulation/parca` POST endpoint only works with SLURM. For K8s, parca runs as part of the Nextflow workflow (not a separate step). Add guard:

```python
if get_job_backend() != "slurm":
    raise HTTPException(501, "Standalone parca not supported for K8s backend (runs in workflow)")
```

### 3c. Base api.yaml Volume Mounts

**File:** `kustomize/base/api.yaml`

The base deployment spec mounts SSH keys and known_hosts. For stanford-test, add a kustomize patch to remove these volume mounts, or make the base conditional.

Simplest: add a JSON patch to the stanford-test overlay that removes the SSH-related mounts:
```yaml
- patch: |-
    - op: remove
      path: /spec/template/spec/containers/0/volumeMounts/1  # SSH key
    - op: remove
      path: /spec/template/spec/volumes/1  # SSH key volume
  target:
    kind: Deployment
    name: api
```
